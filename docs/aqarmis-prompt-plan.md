# خطة برومبت مفصلة لإعادة بناء موقع Aqarmis

## 1. تأسيس المشروع والبنية العامة
- **تهيئة المستودع**: إنشاء مجلدين رئيسيين `backend/` و`frontend/` داخل المستودع مع ملفات README مختصرة توضّح بنية كل جزء ومتطلبات التشغيل الأولية.
- **الواجهة الخلفية (Node.js + Express + MongoDB)**:
  - استخدام TypeScript مع ESLint وPrettier لفرض معايير الكود.
  - تهيئة بيئة خدمات مصغّرة (أو على الأقل فصل وحدات رئيسية مثل المصادقة، إدارة العقود، إدارة الفوترة) باستخدام مجلدات `services/`, `modules/`, `routes/`, `controllers/`, `schemas/`.
  - إعداد اتصال MongoDB عبر Mongoose مع فصل نماذج البيانات لكل كيان رئيسي.
  - تطبيق نظام مصادقة JWT مع إدارة الأدوار (admin، sales، viewer، accountant، lawyer، user) بالاعتماد على Middleware للتحقق من الصلاحيات.
  - إنشاء طبقة تكوين (Config) تقرأ متغيرات البيئة: مفاتيح JWT، عناوين قواعد البيانات، مفاتيح التكامل مع بوابات الدفع وخدمات البريد.
  - إعداد أدوات الإختبار (Jest أو Vitest) مع تغطية أساسية لوحدات المصادقة ونقاط النهاية الحرجة.
- **الواجهة الأمامية (React + Vite + Tailwind CSS)**:
  - تفعيل TypeScript وPostCSS مع Tailwind وتكوين Theme يدعم RTL وLTR.
  - إضافة نظام ترجمة (i18next أو Lingui) يدعم العربية والإنجليزية مع ملفات موارد منفصلة.
  - تهيئة بنية مجلدات واضحة: `components/`, `pages/`, `layouts/`, `hooks/`, `services/`, `store/`, `types/`.
  - إعداد تكامل مع نظام الحالة (Redux Toolkit أو Zustand) لتتبع بيانات المستخدم واللغة.
  - إنشاء ملف إعدادات (`config.ts`) لتخزين نقاط الاتصال بالواجهة الخلفية ومسارات إعادة التوجيه.
- **بيئة التطوير والتشغيل**:
  - ملف `.env.example` يوضح المتغيرات المطلوبة لكل من الواجهة الأمامية والخلفية.
  - إعداد أوامر npm لتشغيل التطوير والبناء والنشر لكل جزء.
  - التفكير المبكر في Docker Compose لتهيئة الخدمات (API، قاعدة البيانات، بوابات التكامل التجريبية).

## 2. الصفحة الرئيسية (Home)
- تصميم صفحة هبوط متعددة الأقسام مع دعم التبديل الفوري بين العربية والإنجليزية.
- عناصر أساسية: قسم البطل، قسم المميزات، قسم لماذا نحن، قسم الأقسام، شريط بحث متقدم، أزرار التسجيل/الدخول، وزر المساعد الذكي.
- تفعيل استعلامات البحث عبر API لإظهار نتائج ديناميكية مع إمكانية دمج خريطة تفاعلية (Google Maps أو بديل محلي).
- إضافة عدادات إحصائية متحركة ونموذج اتصال سريع.

## 3. نظام التوثيق (Login / Register / Verification)
- صفحات تسجيل الدخول، إنشاء حساب، إعادة الإرسال والتحقق من OTP، نسيان كلمة المرور وإعادة التعيين.
- التحقق من صحة المدخلات على الواجهة الأمامية والخلفية مع رسائل واضحة.
- دعم تسجيل الدخول الاجتماعي (Google، Facebook) وإعادة التوجيه إلى `/oauth-success`.
- دمج reCAPTCHA، مؤشر قوة كلمة المرور، ونموذج تسجيل مختصر للعملاء المحتملين.

## 4. لوحة التحكم (Dashboard)
- لوحة معلومات للأدوار المخوّلة تتضمن بطاقات إحصائية، رسوم بيانية تفاعلية، قائمة مهام/تنبيهات، واختصارات لإنشاء كيان جديد.
- استكشاف إضافة تحليلات ذكاء أعمال (BI) وتخصيص عناصر اللوحة لكل مستخدم.

## 5. إدارة المستخدمين (Users)
- صفحة خاصة بالمشرفين تعرض جدول المستخدمين مع وظائف الإضافة، التعديل، الحذف، إعادة تعيين كلمة المرور.
- توفير البحث، التصفية، التصدير إلى CSV/Excel، وإشعارات عند إضافة أو تعديل مستخدم.
- إمكانية ربط المستخدم بفروع أو أقسام.

## 6. إدارة العملاء (Customers)
- قائمة العملاء مع حالة (حالي/محتمل)، وحدة العدادات المرتبطة، ومصدر العميل.
- نموذج إضافة وتحرير عميل مع سجل تفاعل متكامل.
- صفحة تفاصيل تعرض الأنشطة، العقارات، العقود المرتبطة، والتكامل مع البريد والواتساب.
- التفكير في خوارزمية تصنيف العملاء المحتملين وإرفاق المستندات بأمان.

## 7. إدارة العقارات (Properties)
- قائمة العقارات ببيانات أساسية وصور مصغرة.
- نموذج إضافة عقار مع دعم رفع صور متعددة وتحديد الموقع على الخريطة.
- صفحة تفاصيل العقار مع معرض صور، خريطة تفاعلية، الوحدات والعقود المرتبطة.
- خيارات التعديل والحذف، ودعم الوسوم (Tags) واقتراح السعر عبر الذكاء الاصطناعي.

## 8. إدارة الوحدات (Units)
- عرض الوحدات لكل عقار مع حالة الإشغال والأسعار.
- نموذج إضافة وحدة مع ربطها بعقار محدد.
- تفاصيل الوحدة، جدول الصيانة، وربط بمواعيد المعاينة.
- وظائف التعديل والحذف.

## 9. إدارة العقود (Contracts)
- قائمة العقود بمعلومات أساسية وحالات مختلفة.
- نماذج إنشاء عقود بيع وإيجار مع اختلاف الحقول والدفع.
- صفحة تفاصيل العقد مع جدول الدفعات، تحميل PDF، إشعارات انتهاء، وخيارات التعديل والإلغاء.
- دراسة دمج التوقيع الإلكتروني وتحليل المخاطر.

## 10. إدارة الفواتير (Invoices)
- قائمة الفواتير مع حالات الدفع والربط بالعقود.
- نماذج إنشاء الفواتير مع جدول دفعات وخيارات الدفع الإلكتروني (فوري، محافظ محلية).
- تفاصيل الفاتورة، إرسال عبر البريد/الواتساب، تنزيل PDF، والتحرير/الحذف.
- تفعيل إشعارات الدفع والتكامل مع أنظمة المحاسبة.

## 11. المهام والتقويم (Tasks & Calendar)
- قوائم مهام مع حالات متعددة ودعم لوحة كانبان.
- نموذج إضافة وتحرير مهمة مع تحديد الأولوية والمسؤول.
- تقويم شهري/أسبوعي مع إمكانية السحب والإفلات.
- تكامل Google Calendar/Outlook وإشعارات تنبيه.

## 12. نظام الإشعارات (Notifications)
- واجهة لإدارة الإشعارات مع تمييز المقروء وغير المقروء.
- إرسال إشعارات فورية عبر WebSockets أو FCM حول الأحداث المهمة.
- إعدادات لتخصيص قنوات الإشعارات (بريد، تطبيق، هاتف، واتساب).
- توفير ملخصات يومية/أسبوعية قابلة للتخصيص.

## 13. إعدادات المستخدم (Settings & Theme)
- صفحة إعدادات لتحديث البيانات الشخصية، كلمة المرور، اللغة، الثيم، وإعدادات الإشعارات.
- خيار حذف الحساب بعد التحقق وربط الإعدادات بميزة المصادقة الثنائية.
- سجل آخر الأنشطة وتواريخ تسجيل الدخول.

## 14. الدردشة مع الذكاء الاصطناعي (AI Chat)
- نافذة دردشة عائمة متاحة في كل الصفحات مع دعم العربية والإنجليزية.
- تكامل مع API لنموذج لغوي لتقديم إجابات سياقية وربطها بالبيانات الداخلية.
- حفظ السجل في قاعدة البيانات مع ربطه بالمستخدم وإظهار تاريخ الدردشات.
- دعم رفع الملفات، تحليلها، وتقديم تلخيصات ذكية للمحادثات.

## 15. سجل المحادثات (Chat History)
- صفحة تعرض الجلسات السابقة مع خيارات البحث، الفرز، الفتح، الحذف، وتنزيل المحادثات.
- عرض إحصائيات عدد المحادثات ومتوسط طولها، ودعم فلترة حسب نوع الاستفسار.

## 16. الملف الشخصي وميزات المصادقة المتقدمة
- صفحة ملف شخصي مع تحديث البيانات، رفع صورة، ربط حسابات خارجية، وإدارة الجلسات.
- عرض سجل الدخول، دعم توثيق الهوية بالمستندات، وإغلاق الجلسات المريبة.

## 17. التصميم والجودة وتجربة المستخدم
- التزام بتصميم متجاوب يدعم RTL/LTR والوضعين الداكن والفاتح.
- استخدام أنيميشن خفيفة تحافظ على الأداء وتلبي معايير الوصول (WCAG).
- توفير خريطة موقع داخلية وبحث شامل في أعلى الصفحات.

## 18. الأمان والامتثال
- فرض تشفير الاتصالات (SSL/TLS) وتخزين كلمات المرور بـ bcrypt.
- حماية API ضد XSS, CSRF, Rate Limiting، وتفعيل RBAC.
- حفظ السجلات ومراقبة الأنشطة مع الالتزام بالقوانين المحلية والدولية (قانون حماية البيانات المصري، GDPR).
- تنفيذ نسخ احتياطية دورية واختبارات أمنية ووحدات.

## 19. التوسع والتكاملات
- تصميم API REST/GraphQL مع نظام Versioning ودعم Webhooks.
- تكامل مع بوابات الدفع المحلية والدولية، خدمات الرسائل القصيرة والبريد، ومنصات التسويق.
- دعم الحاويات (Docker) والاستضافة السحابية، والنظر في Serverless للمهام الخلفية.
- التفكير في تكاملات IoT مستقبلية مثل العدادات الذكية.

## 20. الذكاء الاصطناعي والتحليلات
- تنفيذ نماذج لتصنيف العملاء المحتملين، توقع الأسعار، وتقديم توصيات وإشعارات ذكية في لوحة التحكم.
- تحليلات تنبؤية للمبيعات والإيرادات مع تقارير تفصيلية قابلة للتصدير.
- استخدام معالجة اللغة الطبيعية لتلخيص المحادثات وتحويلها إلى مهام.
- دمج الرؤية الحاسوبية لتحليل صور العقارات.

## 21. متطلبات السوق المصري
- دعم الجنيه المصري مع إمكانية تغيير العملة.
- الالتزام بقوانين التسجيل العقاري وإدراج البنود الملائمة في العقود.
- تكامل مع منصات MLS المحلية (مثل Egypt MLS وArab MLS) وتوقيع العقود عبر التوثيق الرقمي.
- دعم بوابات الدفع المحلية مثل فوري وأمان والمحافظ الإلكترونية.

## 22. خطة التنفيذ والتكامل المستمر
- تقسيم العمل إلى سباقات (Sprints) تبدأ بالبنية الأساسية ثم الوحدات الحرجة.
- تطبيق CI/CD باستخدام GitHub Actions أو GitLab CI لتشغيل الاختبارات والفحوصات الآلية.
- إعداد بيئات متعددة (Development, Staging, Production) مع مراقبة الأداء باستخدام أدوات مثل New Relic أو Datadog.
- تنظيم جلسات مراجعة دورية لضمان جودة التنفيذ والالتزام بالخطة.
